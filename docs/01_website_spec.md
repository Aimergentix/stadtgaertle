## 3. Asset rules

| Asset type       | Location              | Referenced as                    |
|------------------|-----------------------|----------------------------------|
| Images           | `public/images/`      | absolute path `/images/name.ext` |
| Visible copy     | `src/config/site-content.json` | imported in component, never inlined |
| Navigation labels| `src/config/navigation.json`   | imported in component, never inlined |
| Outbound URLs    | `src/config/services-router.ts`| imported in component, never inlined |
| TypeScript types | `src/types/`          | named exports, one file per schema |

Images are never imported with `import heroImg from './hero.jpg'`. They are always
referenced as string paths: `<img src="/images/hero.webp" alt={content.hero.imageAlt} />`.

## 4. SOA decoupling contract (HACE §6)

This website is a **Content Display Shell**. It has no coupling to any authenticated or
stateful service.

**Hard constraints — any violation is a structural error:**
- Zero backend logic in `/frontend/` code.
- Zero authentication state or session management.
- Zero database drivers or API client SDKs.
- All member-facing actions ("Login", "Dashboard") are plain `<a>` tags.
  Their `href` values come exclusively from `src/config/services-router.ts`.
- A separate authentication portal lives at a future subdomain (not yet defined).

---

## 5. Non-goals (explicit, permanent for v1)

- Server-side rendering (SSR).
- API routes or any server-side logic.
- Database access of any kind.
- Authentication, sessions, or user accounts.
- Form processing with persistence (contact forms submit via `mailto:` only).
- Dark mode.
- Internationalisation (i18n). The site is German-only in v1.
- Interactive mapping. PlotMapSection is a static image with text labels.
